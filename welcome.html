안녕하세요 Lazypic 입니다.
우리가 이름지은 Lazypic에는 숨겨진 의미가 있습니다.

"천천히 꿈을 그리자."
"미래에 이로움을 줄 수 있는것을 만들고 베풀자."

프로젝트의 진행 속도보다 긍정적인 에너지를 가진 방향성이 더 중요합니다.
미래에 여러분이 작성하는 소프트웨어가 콘텐츠를 제작하는 사람들에게 이로운 방향이 되었으면 좋겠습니다.

소 개

Lazypic은 콘텐츠 제작지원 소프트웨어를 개발합니다.
폐쇠적인 방식이 아닌, 웹과 오픈소스 기반위에서 집단지성을 발휘하여 협업합니다.
원하는 프로젝트를 자유롭게 생성하고, 공유하며 함께 이루어나갑니다.
결과보다는 진행 과정의 즐거움을 중요시합니다.
프로젝트를 진행하는 모든 구성원의 관심 분야(기초 학문, 응용기술 등)를 공유하고, 함께 공부하며 진행해갑니다.

프로젝트 생성

스스로 프로젝트를 생성하거나 다른 사람의 프로젝트에 참여할 수 있습니다.
친구, 개인, 기업, 재단등과 계약을 통해 오픈소스 프로젝트를 생성하거나 콜라보레이션을 하는 것도 가능합니다.
다양한 분야의 사람들과 프로젝트를 협업하고 싶다면 프로젝트를 기획하고 전파하세요!

소프트웨어 사용

상용소프트웨어 사용을 지양합니다.
오픈소스 툴을 사용하거나 필요한 툴을 만들고,
협업을 통해 오픈소스 도구를 만들고 세상에 출시하는 것을 지향합니다.

수익계산과 분배

운영비, 미래 준비금을 제외한 나머지 수익에 대해
프로젝트 참여자의 기여도에 따라 분배합니다.
프로젝트의 경우 Git-hours툴과 PR, 그래프 데이터를 활용하여 수익비율을 계산합니다.
수익 비율을 계산하기 힘든 경우(ex.콘텐츠), 프로젝트 참여자끼리 회의를 통해 수익분배를 결정합니다.

교육문서

Lazypic 내부 교육문서도 하나의 개발툴로 보고 Github에 작성됩니다.
Lazypic의 운용 메뉴얼은 교육이 필요한 개인, 학교, 재단 및 기업의 2차 교육 자료로 활용됩니다.

개 발

Lazypic 은 Go를 활용하여 많은 개발이 이루어지고 있습니다.
배포의 편의성이 크게 향상되며, OS 의존성이 낮아지기 때문입니다.

우리는 다음과 같이 역할을 나누어 프로젝트를 진행합니다.
Maintainer: 리포지터리 담당 관리자
Committer: 코드를 작성, 수정, 리뷰
Contributors : 쉬운 이슈, 오타등을 처리

BSD 라이센스

대부분의 프로젝트는 BSD 라이센스 정책을 따르며 진행하고 있습니다.
여러분이 작성한 코드는 언제 어디서나 자유롭게 활용할 수 있습니다.
또한 개발자 스스로가 작성한 코드는 이력과 포트폴리오, 기업 자료 등으로 자유롭게 활용 가능합니다.
BSD 라이센스 특성상, 추후 클로즈드 소스로 변환하여 사용하더라도 문제없습니다.
단, 소프트웨어를 개발할 때 여러분이 잊지말아야 할 점은 내가 하는 행동이 콘텐츠 제작 시장을 이롭게 하는 행동인지 생각해 보는 것 입니다.

클라우드 지원

개인, 테스트 프로젝트, 기업 모두 안정적인 소프트웨어, 서버환경을 추구합니다.
우리는 AWS 및 테스트 코드 작성을 통해서 이 부분을 해결합니다.
클라우드 자원이 필요하다면 아래 메일로 AWS 키를 요청하세요.
요청메일: hello@lazypic.org

Lazypic은 Amazon Web Service에 인프라를 구축해서 프로젝트를 진행합니다.
프로젝트가 다른 국가에서 진행되는 경우, 재난, 국제적으로 프로젝트 이주시 해당 국가로 데이터를 이전합니다.

프로젝트 종료

발전 가능성이 없는 프로젝트의 경우에는 Legacycode 로 프로젝트를 이동합니다.
사용했던 클라우드 리소스는 제거합니다.
백업이 필요한 데이터는 AWS Glacier Deep Archive 로 보관합니다.
