Lazypic은 대부분 소프트웨어 개발에 BSD 라이선스 정책을 적용하여 개발합니다.

제작된 소프트웨어는 Github를 통해 오픈소스로 공유합니다.

Most of softwares of Lazypic is developed under the BSD license policy.

The created software is distributed as open source through Github.

한글 English

BSD License

BSD 라이선스는 자유 소프트웨어 저작권의 한 종류입니다.
해당 소프트웨어 코드를 누구나 수정할 수 있고 수정한 것을 제한 없이 배포할 수 있습니다.
BSD 라이선스로 개발된 소프트웨어와 코드를 회사에 들여서 내부에 사용해도 법적으로 문제가 없습니다.
기업이 코드를 공개하지 않고 상용 소프트웨어로 전환하여 사용할 수 있습니다.
반대로 기업 내부에서 만들어진 소프트웨어 라고 하더라도 BSD 라이선스 정책에 따라 만들어진 소프트웨어라면,
코드를 작성한 개발자도 외부에 자유롭게 활용할 수 있습니다.

The BSD license is a kind of free software copyright.
Anyone can modify and distribute the source code with minimal restrictions.
There is no risk of legal conflict when companies use software and code developed under the BSD License.
Company can switch open-source software to commercial software without disclosing its source code.
But it also means that even if the software was created in-house, developer of the software can use it freely(outside of the company), if it was created under the BSD license.

Lazypic이 BSD 라이선스를 사용하는 이유

Why Lazypic uses a BSD license.

기업에서 제작된 소프트웨어의 수명은 기업의 수명과 함께하며, 일반적인 기업의 평균 수명은 15년 정도입니다.
Lazypic도 이 사실을 잘 알고 있습니다.
우리의 소프트웨어의 수명을 늘리기 위해 좀 더 자유로운 형태의 라이선스를 채택하고 개발하기로 했습니다.
개발자의 실력이 향상하여 다른 곳에서 자신의 역할을 수행하더라도 사회에 가치 있는 소프트웨어라면 지속해서 개발되기를 원합니다.
우리가 만드는 소프트웨어가 Lazypic 내부에서만이 아니라 사회 곳곳에서 콘텐츠 제작에 활용되기를 희망합니다.
또한 BSD 라이선스의 자유로운 정책으로 인해 훌륭한 개발자를 만나 좋은 소프트웨어로 거듭날 수 있는 가능성을 열어둘 수 있습니다.
또 다른 자유 소프트웨어 저작권인 GPL 라이선스는 너무 제약이 강하기 때문에 이 라이선스를 사용하는 것만으로도 많은 기업이 솔루션을 도입하기 꺼립니다.
Lazypic은 1인 콘텐츠 제작자, 학교, 재단, 기업과 자유로운 협업을 위해서 BSD 라이선스 정책을 따릅니다.

The lifespan of software produced by an enterprise is in line with the lifespan of an enterprise,
and the average life expectancy of a typical company is about 15 years.
Lazypic knows this too.
To extend the life of our software, we decided to adopt and develop a more liberal license.
As developers improve their skills, they want to continue to develop software that is valuable to society, even if they play their role elsewhere.
We hope that the software we create will be used for content creation not just within Lazypic, but across society.
Also, the liberal policy of the BSD license allows you to open the possibility of meeting good developers and becoming good software.
Another free software copyright, the GPL license, is so restrictive that many companies are reluctant to adopt the solution just by using it.
Lazypic follows the BSD Licensing Policy for collaboration with content creators, schools, foundations, and businesses.

책 임

Responsibility

BSD 라이선스가 적용된 코드는 자유롭게 사용하는 대신,
사용자에게 발생하는 모든 리스크와 손해에 대해서도 사용자가 책임을 지게 됩니다.
사용자 스스로 코드를 이해하고 사용해야 합니다.

If you used to BSD licensed code and software,
You are responsible for all risks to you.
You must understand and use the code yourself.

기업의 장점

Advantage of corporate

사내 개발자는 초기부터 많은 곳에서 소프트웨어를 적용할 수 있다고 생각하고 소프트웨어를 설계하게 됩니다.
어쩌면 평생 사용하게 될 수도 있습니다. 이러한 특징 때문에 기업 입장에서는 양질의 소프트웨어를 얻을 수 있는 확률이 높습니다.
또한 회사에서 진행하는 프로젝트를 오픈소스로 진행하면 관련 개발자를 구하기 쉬워집니다.
다른 곳에서 만들어진 오픈소스 라이선스 소프트웨어도 자유롭게 도입 및 활용이 가능하기 때문에 전체 개발 기간이 줄어듭니다.

In-house developers design software because they think it can be applied in many places from the beginning.
Maybe you will use it forever. Because of these features, companies are more likely to get quality software.
Also, open-source projects in your company can help you find relevant developers.
Open-source licensed software created elsewhere can be freely adopted and utilized, reducing overall development time.

기업의 단점

Disadvantages of corporate

BSD 라이선스도 특허, 소프트웨어 등록이 가능합니다.
하지만, 보통 오픈소스 소프트웨어를 무형자산으로 등록하지 않습니다.
시장에서 기술 선점을 위한 전략을 취할 때는 오픈소스로 진행할 프로젝트와 비공개로 진행할 프로젝트를 분리할 필요가 있습니다.

A BSD license is also available for patent and software registration.
However, open-source software is not normally registered as an intangible asset.
When taking a strategy for technology preoccupation in the market,
It is necessary to separate the project that is going to be open source from that which is going to be private.

개발자의 장점

Advantage of Developers

개발을 중단만 하지 않는다면 일생에 걸쳐서 사용할 수 있는 소프트웨어, 커리어, 포트폴리오를 구축할 수 있습니다.

If you don't stop development, you can build software, careers, and portfolios in your lifetime.

개발자의 단점

Disadvantages of Developers

자신의 실력이 명확하게 드러납니다.
하지만 수 많은 사람에게 코드 리뷰를 받을 수 있는 장점이 있습니다.

Your skills are clearly visible.
But, you have the advantage of receiving code reviews from many people.

특 허

Patent

공개 라이선스로 배포하더라도 특허 등록이나 특허권 유지가 가능합니다.
공개 소프트웨어 라이선스는 소스 코드의 자유로운 사용을 위해서 마련되었기 때문에,
많은 공개 소프트웨어 라이선스가 특허권자의 특허권 주장을 금지하는 정책을 취하고 있습니다.
공개 소프트웨어로 배포하되 특허권 주장을 원한다면 특허와 관련한 내용이 없는 라이선스를 선택하는 것이 좋습니다.
대표적으로 특허 관련 내용이 없는 라이선스로는 BSD, MIT 라이선스가 있습니다.

You can register a patent or maintain a patent even if you distribute it under a public license.
Since open software licenses are designed for the free use of source code,
Many open software licenses have a policy that prohibits patent holders from claiming patents.
If you distribute as open software but wish to claim a patent, we recommend that you select a license that does not contain any patent content.
Typical licenses without patents include BSD and MIT licenses.

개발자가 알아야할 사항

What developers need to know

다른 곳의 소스 코드를 사용할 때 소스 코드의 라이선스와 정책을 체크하고 사용하세요.
오픈소스를 충분히 검토하고 사용하는 것을 추천합니다.

When using the source code elsewhere, check and use the licenses and policies of the source code.
We recommend that you thoroughly review and use open source.
